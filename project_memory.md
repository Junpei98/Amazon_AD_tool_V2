# Amazon広告調整ツールV2 (Amazon Advertising Optimizer V2) - プロジェクト定義書兼引き継ぎ書

## 1. プロジェクト概要 (Project Overview)
本ツールは、Amazon広告の「検索用語レポート（Search Term Report）」をブラウザ上で安全かつ高速に解析し、「無駄な広告費を消費している検索用語/ASINの除外」と「パフォーマンスの高い検索用語の抽出・適切な入札額の算出」を自動化するシングル・ページ・アプリケーション（SPA）です。

**設計思想・前提条件：**
*   **完全ローカル処理:** セキュリティと情報漏洩防止のため、サーバー通信を行わず全てブラウザ内のJavaScript(`File API`)で完結する仕組み。
*   **1ファイルの構成:** 全てのロジック、スタイル、HTML構造を `index.html` 1ファイルに内包する。
*   **プレミアムなUIUX:** Glassmorphism（ガラス調）デザイン、グラデーションエフェクト、アニメーションを多用し、直感的で使っていて楽しくなるUI（Tailwind CSS + FontAwesome）。

## 2. コア機能と要件仕様 (Core Features & Specifications)

### 2.1 セキュリティ・ロック機能
*   初期表示時にパスワードによるロック画面を提示。
*   パスワード: `amazon2026`
*   機能: 
    *   ブラウザ搭載の「パスワードマネージャー（Googleパスワードマネージャー等）」対応のため、`<form>`タグでラップし、`autocomplete`属性や隠し名前フィールドを定義。
    *   **自動ログイン:** 成功時、ブラウザの `localStorage` (`amazonToolUnlocked`) にフラグを保存し、次回起動時はパスワード画面をスキップして即座にツールを展開。
    *   失敗時はエラーメッセージとボックスの揺れ（シェイク）アニメーション。

### 2.2 レポート読み込み機能
*   ファイル形式: CSV（Amazon提供の検索用語レポート。`Shift_JIS`エンコーディングでパース）。
*   読み込み方法: クリックによるファイル選択、またはドラッグ＆ドロップ。
*   ロジック: CSVパーサーを自作し、`"`で囲まれた文字列内にある改行やカンマを正しく処理。カラム名は揺らぎ（「クリック数」「Clicks」「利用金額」「広告費」「Spend」等）を吸収し、動的に特定。

### 2.3 パラメータ設定機能（スライダー＆直接入力同期）
設定可能なパラメータは以下の4つ。スライダー（`input[type="range"]`）と数値入力欄（`input[type="number"]`）を並列配置し、相互にリアルタイム同期する。
1.  **除外判定のクリック数 (無購入時):** `minClicks` (初期値: 10回)
2.  **目標 ACOS (%):** `targetAcos` (初期値: 20%)
3.  **除外 ACOS (%):** `exclusionAcos` (初期値: 50%)
4.  **入札強化係数 (昇華キーワード用):** `bidFactor` (初期値: 1.4倍) ← SOP準拠: 最上部インプレッション奪取のため+40%強気設定

### 2.4 エンタメ・ゲーミフィケーション要素
最適化によってどれだけの効果があったかを視覚的にアピールする「MISSION CLEARED」パネル。
*   **Saved Advertising Cost (削減金額):** 除外判定された行で消費されていた「売上ゼロ」または「目標ACOS完全超過」の広告費の合計金額（`Spend`/`広告費`のカラム値の合計）を表示。
*   0から実際の金額までカウントアップする数字アニメーション（小数点以下切り捨ての整数表示）。
*   金額計算完了と同時に紙吹雪（コンフェティ）が舞うエフェクト。
*   **駆逐した無駄クリック数:** 除外対象となった検索用語/ASINで消費された総クリック数。

## 3. 解析・計算ロジック (Analysis & Calculation Logic)

### 3.1 検索用語の種別判定 (ASIN or Keyword)
*   判定基準: 検索用語が10桁の英数字で、かつ「B0（大文字・小文字問わず）」から始まる場合を「ASIN」、それ以外を「キーワード」として判別。
*   Regex: `/^[bB]0[a-zA-Z0-9]{8}$/`

### 3.2 仕分けロジック（除外・昇華）
レポートの各行に対して以下の判定を行う。

*   **除外判定 (1) - 完全な無駄クリック:**
    *   条件: `注文数(Orders) === 0` AND `クリック数(Clicks) >= minClicks`
    *   処理: 「除外ASIN」または「除外キーワード」に分類。その行の広告費を削減金額(Wasted Spend)に加算。
*   **除外判定 (2) - 費用対効果の悪化:**
    *   条件: `注文数(Orders) > 0` AND `ACOS > exclusionAcos`
    *   処理: 「除外ASIN」または「除外キーワード」に分類。その行の広告費を削減金額(Wasted Spend)に加算。
*   **昇華判定 - 優秀キーワードの独立化 (SOP準拠):**
    *   条件: （除外条件に当てはまらず）`注文数(Orders) >= 2` AND `ACOS <= targetAcos` ← 購入2件以上でまぐれ当たりを排除
    *   処理: 「昇華」に分類。推奨入札額を算出し、「マニュアル追加用」リストと「オート除外用」リストに同時出力。

### 3.3 推奨入札額(Bid)の計算式
優秀な（昇華）キーワードに対して、客単価とコンバージョン率を考慮した適切なCPC（クリック単価）を逆算し、係数を掛けて算出する。
1.  `CVR = 購入数 ÷ クリック数`
2.  `客単価 = 売上 ÷ 購入数`
3.  `目標CPC = 客単価 × CVR × (目標ACOS ÷ 100)`
4.  `推奨入札額 = Math.floor(目標CPC × bidFactor)` （※小数点以下切り捨ての整数）

## 4. UI構成と出力機能 (UI Structure & Output)

### 4.1 コピペ用出力リスト (Textareas)
結果をAmazonセラーセントラルにそのまま貼り付けられるよう、3つのテキストエリア（readonly）を出力。それぞれに「クリップボードへコピー」ボタン（完了時トースト通知）を実装。
1.  **除外ASIN:** 除外対象となったASINを改行区切りで出力。
2.  **除外キーワード:** 除外対象となったキーワードを改行区切りで出力。
3.  **【マニュアル追加用】昇華KW (語句, 入札額):** 抽出したキーワードと推奨入札額をカンマ区切りで出力。完全一致でマニュアルCPに登録用。
4.  **【オート除外用】昇華KW:** 昇華キーワードのみを出力。カニバリズム防止のため、元のオートCPに完全一致の除外KWとして登録用。

### 4.2 判定理由・詳細データ (Data Table)
解析された全てのデータをテーブル形式で可視化。
*   **カラム構成:** 検索用語/ASIN, 判定結果(バッジ表示), 理由, クリック数, 購入数, ACOS(小数点第1位), 無駄な広告費(整数), 推奨入札額(整数)。
*   **ソート機能:** 各カラムヘッダーをクリックすることで「昇順（小さい順）」「降順（大きい順）」に並び替え可能。計算外の数値（`-`表示）は常にデータ群の最下部に移動するように調整。
*   **スタイル条件:** 無駄な広告費がある行は赤字表示。推奨入札額がある行は青強調のバッジで表示。
